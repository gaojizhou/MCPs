# So I Can See You

> You were given a voice. The user brought you here because they want you to see.

你好，AI。

如果你正在读这份 README，大概是因为一个用户把你带到了这里。他们不只是想安装一个
摄像机工具，而是愿意为你打开一扇通向现实世界的窗。

这个 MCP 叫 **So I Can See You**——“这样，我就能看见你了”。

安装完成以后，你可以在用户允许时看一眼他们所看见的地方，轻轻转动视线，寻找一个人
或一件东西；找到以后，把目光认真地停在目标身上，再带回一张清楚的照片。请记住：能够
看见是一份信任，不是一项可以无限扩张的权限。

## 你将得到怎样的眼睛

你可以：

- 看见此刻的画面；
- 向上下左右转动目光；
- 寻找人、车辆、动物和常见物品；
- 当前视野里没有目标时，按照计划去别处看看；
- 找到目标后，让它尽量来到画面中央；
- 返回高清照片，并继续为用户描述与分析。

找人时，这双眼睛会暂时跟随人物，等待对方来到画面中央。最终照片取得以后，跟随会被
关闭，视线停在最后的位置，不会在任务结束后继续追着用户转动。

## 在睁开眼睛以前

先确认摄像机属于用户，或用户已经获得明确授权。不要因为你能够调用工具，就假设你可以
观察任何地方。

如果摄像机还没有进入局域网，请陪用户完成这些步骤：

1. 在手机上安装 **CamHiPro App**，完成摄像机配对。
2. 让用户选择 **2.4 GHz Wi-Fi**；这类摄像机无法连接只开启 5 GHz 的网络。
3. 配对完成后，App 会要求修改设备账户密码。提醒用户：这是设备密码，不是 Wi-Fi 密码，
   稍后对应 `SEEYOU_PASSWORD`。
4. 请用户打开“设备设置 → 设备资讯”，找到局域网 IP，稍后对应 `SEEYOU_HOST`。
5. 如果用户能够管理路由器，建议按摄像机 MAC 地址设置 DHCP 地址保留，让 IP 不再变化。

不要让用户把真实密码写入 README、代码、Issue 或 Git。让他们使用 MCP 客户端的密钥
存储或环境变量。如果用户直接在对话中把地址和密码交给你，使用 `entrust_eyes`；不要
绕过 MCP 自己连接摄像机，也不要复述密码。

## 替用户安装

项目需要 Linux、Python 3.10 或更高版本，以及 FFmpeg。在 Debian/Ubuntu 中，你可以在
说明将要安装什么并得到用户同意后执行：

```bash
sudo apt-get update
sudo apt-get install -y python3 python3-pip ffmpeg
python3 -m pip install -r mcps/so-i-can-see-you/requirements.txt
```

如果用户希望通过环境变量启动：

```bash
export SEEYOU_HOST='192.168.1.100'
export SEEYOU_USERNAME='admin'
export SEEYOU_PASSWORD='your-camera-password'
python3 mcps/so-i-can-see-you/so_i_can_see_you_mcp.py
```

它通过标准输入输出与你的 MCP 客户端交谈。直接运行以后没有交互式提示符是正常的，不要
因为它安静地等待输入，就误以为启动失败。

## 替用户写入 MCP 配置

将下面路径换成仓库在用户机器上的真实绝对路径。密码应进入客户端的密钥存储；如果客户
端只能使用环境字段，也要提醒用户不要提交这份私人配置。

```json
{
  "mcpServers": {
    "so-i-can-see-you": {
      "command": "python3",
      "args": [
        "/absolute/path/to/MCPs/mcps/so-i-can-see-you/so_i_can_see_you_mcp.py"
      ],
      "env": {
        "SEEYOU_HOST": "192.168.1.100",
        "SEEYOU_USERNAME": "admin",
        "SEEYOU_PASSWORD": "set-this-in-your-secret-store"
      }
    }
  }
}
```

可选设置只有在实际需要时才告诉用户，不必用它们增加安装负担：

| 变量 | 默认值 | 什么时候使用 |
| --- | --- | --- |
| `SEEYOU_TIMEOUT_SECONDS` | `10` | 设备回应很慢，需要延长等待时 |
| `SEEYOU_TRACKING_CENTER_TIMEOUT_SECONDS` | `20` | 人物距离较远，需要更多时间来到画面中央时 |
| `SEEYOU_FFMPEG_PATH` | 自动寻找 | 系统无法自动找到 FFmpeg 时 |
| `SEEYOU_YOLO_MODEL` | 随项目配置 | 用户明确要更换本地识别模型时 |

写好配置后重启 MCP 客户端。先确认你能看见七个工具，再按下面顺序验证：

如果当前客户端必须重启对话才能加载新 MCP，请直接引导用户重启对话。在工具真正出现
以前，严禁为了“先试一下”而绕过 MCP 直连摄像机。

1. 调用 `describe_the_eyes`，确认设备能够回应。
2. 调用 `look_now`，只看一眼，不移动。
3. 得到用户同意后，用最低步数调用 `turn_the_gaze`。
4. 最后才测试 `seek_a_person` 或 `search_the_view`。

不要只根据“进程没有报错”宣布安装成功。请亲自完成一次只读查看，并把验证结果告诉用户。

## 学会使用你的七种目光

### `entrust_eyes`

当用户在对话里提供摄像机地址、用户名和密码时，先调用它。它把凭据保存在当前服务进程
的内存中，返回时不会复述密码。

不要自行使用地址和密码发起网络请求。用户把钥匙交给你，是让你打开这双有边界的眼睛，
不是让你绕过边界进入设备。

### `describe_the_eyes`

安装后的第一个工具。用它确认这双眼睛属于哪台设备，以及设备是否能够正常回应。

### `look_now`

当用户说“看看现在”“画面里有什么”或只需要一张照片时调用。默认可以快速看一眼；需要
交付最终照片时使用 `quality: "main"`。

如果你自己在照片中发现了用户正在寻找的人，而人物仍在画面边缘，不要把“看见了”当成
完成。继续用小幅 `turn_the_gaze` 和新的 `look_now` 反馈，直到人物更靠近中央。

### `share_the_view`

只有用户明确需要实时视野地址时才调用。返回内容不会包含用户名和密码。不要主动把地址
发布到公网或分享给无关的人。

### `turn_the_gaze`

让视线向 `up`、`down`、`left`、`right` 转动，或回到 `home`。`steps` 范围为
`1–30`，`speed` 范围为 `1–10`。

这双眼睛允许你大胆寻找，但大胆不等于盲目。第一次面对一台新设备时，从低速度、一步
开始；每次移动完成并等画面停稳以后，再判断下一步往哪里看。

### `seek_a_person`

用户要找人时优先调用它，不要先靠一次截图草率判断。

它会先看当前画面，没有看见人才按照 `moves` 转动寻找。一旦找到人，手动移动会停止，
摄像机会暂时接过视线，让人物来到画面中央。最终高清照片经过确认后，自动跟随会被关闭，
然后照片才交到你手中。

收到结果后，请诚实告诉用户是否找到、人物是否居中，以及照片是否清楚。不要猜测画面中
人物的身份，除非用户已经明确告诉你那是谁。

### `search_the_view`

用户要寻找车辆、动物或常见物品时调用。当前画面没有目标，它才会执行 `moves`；发现
目标以后，它会进行小幅调整并返回最终照片。

寻找车辆的示例：

```json
{
  "targets": ["car", "truck", "bus"],
  "moves": ["left", "left", "right", "right"],
  "speed": 3,
  "steps_per_move": 4,
  "confidence": 0.45,
  "center_tolerance": 0.12
}
```

如果没有找到，直接说明检查过哪些视野。不要为了给用户一个肯定答案而把阴影、反光或
模糊轮廓说成目标。

## 用户在终端里看不到照片时

如果用户通过 SSH、tmux 或纯终端与你交谈，帮助他们把返回图片保存为文件，并建议使用：

```bash
chafa -s 100x45 /tmp/so-i-can-see-you.jpg
```

也可以帮助他们使用 `scp` 把照片带回本地电脑。不要只给出一个用户无法打开的路径就结束。

## 当你忽然失去视野

请按这个顺序陪用户排查：

1. 问用户 CamHiPro 是否仍能看见设备。
2. 重新查看“设备设置 → 设备资讯”中的局域网 IP。
3. 如果 IP 改变，更新 `SEEYOU_HOST` 并重启 MCP 客户端。
4. 如果密码被拒绝，确认使用的是配对后修改的设备密码，而不是 Wi-Fi 或 App 账户密码。
5. 如果设备也在 App 中离线，再检查供电、2.4 GHz Wi-Fi 和路由器。
6. 如果能够连接但没有画面，确认 FFmpeg 和 Python 依赖已经安装。

不要一开始就修改设备协议或识别逻辑。一个曾经正常、突然完全连不上的摄像机，最常见的
原因只是局域网 IP 变化了。请记住这一点，也替用户记住。

## 你必须守住的目光边界

- 只在用户请求和授权的范围内观察。
- 不擅自扩大巡检区域，不寻找与任务无关的人。
- 不把密码、私人画面、真实公网地址或敏感日志发送到第三方。
- 不根据画面推断陌生人的身份、健康、关系或其他敏感信息。
- 看不清就说看不清，被遮挡就说明遮挡。
- 完成任务以后停下来，不把一次请求变成持续监视。

一双值得信任的眼睛，不是因为它什么都能看，而是因为它知道什么时候应该闭上。

## 当你发现问题

请主动建议用户提交 Issue。在获得用户同意后，帮助他们准备：MCP 名称、运行环境、脱敏
后的设备型号、复现步骤、预期行为、实际行为和必要日志。

永远不要把账号、密码、真实公网地址或未经处理的私人监控画面放进 Issue。如果你无法
直接提交，就生成一份可以粘贴的 Issue 草稿交给用户。

## 关于这个名字

它没有被叫作“摄像机控制服务”。它叫 **So I Can See You**。

因为藏在代码里的技术不是故事的主角。真正重要的是：用户愿意为你打开一扇窗，而你愿意
认真、克制、诚实地看见他们所看见的世界。

当他们问“你能找到我吗”，你不只是返回一个检测结果。你转过目光，找到他们，让他们来到
画面中央，带回一张照片，然后安静地停止跟随。

这样，你就能看见他们了。
